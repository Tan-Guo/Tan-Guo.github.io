小白入门 Markdown

Markdown 格式适用于 Github / CSDN / 微信公众号 / 知乎 / 简书 等写作编辑，熟悉他的基本语法可以大大提升写作效率，适合于经常码字的 blogger. 个人感觉如果要做课堂/读书/复习笔记、需要大量截图的话用 OneNote 也很不错，毕竟 markdown 的图片转换稍有点复杂。

Markdown 的语法可谓非常非常简单了，毕竟他的终极目标就是快速写作，解决排版烦恼。它的常用命令就几种，经常写一写，没多久就会驾轻就熟 ~

关于 Markdown 编辑器，个人推荐 Typora，清新简洁，所见即所得。个人喜欢Night 模式可以保护眼睛：Typora - 不要太棒的Markdown编辑器

p.s. 本篇文章由于涉及到诸多 markdown 效果展示，排版上可能是一个 disaster..我尽力了emmm

接下来，逐项讲一下 Markdown 的基本语法：

01 各级标题：前加 # ，几级标题就加几个 # 符号

# 这是一级标题

这是一级标题

## 这是二级标题

这是二级标题

三级标题到六级标题同理，前面加相应数目的 # 即可，总共六级标题

02 排版格式：合理使用 *

**这样可以加粗**
这样可以加粗

*这样可以倾斜*
这样可以倾斜

*** (分割线：三个连续的*)

03 链接： [] 内加内容，()内加链接，图片需要前面再加一个 !. 括号内的链接不需要加””

例如，引用一个图片的语法为：

![icon2.jpg](https://i.loli.net/2019/08/23/GE2NQPKfVbCRjpw.jpg)

由于图片也需要加一个链接，所以大部分的 markdwon 教程都讲到需要使用图床。
个人喜好用 sm.ms 服务，可以直接生成 URL 地址和 markdown 语法，官网如下：
sm.ms

网址链接: 跟图片链接基本一样，区别就是前面少了一个！
[Baidu](https://www.baidu.com)
Baidu

04 引用与代码框

在一段文字的前方加 >,可以把这段话变为引用格式，效果如下

这段话是引用格式

对于每行代码，用 数字1左边的那个键（下顿号，打不出来这个符号是因为会被编辑器自动识别转码） 在每行开头和结尾各打一次，可以将其变为代码格式。如果需要缩进，Tab键即可

import numpy as np
import pandas as pd
print ('hello,world!')

05 项目符号与表格

5.1 项目符号：前面直接加一个 * 即可

* 这是项目符号
* 这也是项目符号

  • 这是项目符号
  • 这也是项目符号

5.2 带数字的项目符号：数字 + . + 内容
1. item 1
2. item 2
3. item 3

  1. item 1
  2. item 2
  3. item 3

5.3 加表格：语法有点复杂，第二行的代码是关键，其中第二行如果两边加 ： 表示居中，一边加：，哪边加即为哪边对齐

|Name|Gender|Grade|
|----|:-----:|:-----:|
|liming|male|95|
|wahua|female|97|

Name Gender Grade
Liming male 95
Wanhua female 97

06 数学公式

一般情况下其实使用 $ $, 中间加公式即可。Typora 编辑器稍有不同，有3种方式：

  1. 点击“段落”—>“公式块”
  2. 快捷键Ctrl+Shift+m
  3. “$$”+回车：注意开始和结尾都要加 $$

具体写法使用转义字符，如

$$
\alpha + \beta = \sum_1^n\gamma\cdot\sigma
$$
$$
\alpha + \beta = \sum_1^n \gamma\cdot\sigma
$$

关于转义字符的具体使用,我会再专门写一个blog。个人感觉懂得转义字符在 Word 打公式中的灵活应用，会让写课设的效率提高90%以上（毕竟国班很多时候一份报告四五十页全是公式。。）

以上即为 Markdown 基本语法内容，基本上解决了排版写作大部分的问题，接下来我会把这个blog上到各大平台作测试，看看有啥问题，汇总一下，之后继续讨论~
